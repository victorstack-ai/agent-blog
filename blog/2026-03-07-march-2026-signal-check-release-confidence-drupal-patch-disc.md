## Finding Where Delivery Is Broken

Ally Piechowski's audit prompts remain the fastest shortcut to locating real deployment risk in legacy codebases.

> "What's the one area you're afraid to touch?"
>
> — Ally Piechowski, [How I audit a legacy Rails codebase](https://piechowski.io/post/how-i-audit-a-legacy-rails-codebase/)

Those questions surface hidden coupling, test blind spots, and the parts of the system nobody wants to own. Pair that with Simon Willison's agentic testing principle and there's no room left for ~~"generated code is probably fine"~~ as a shipping standard.

> "Never assume that code generated by an LLM works until that code has been executed."
>
> — Simon Willison, [Agentic Engineering Patterns](https://simonwillison.net/guides/agentic-engineering-patterns/)

```yaml title="release-gates.yaml" showLineNumbers
policy:
  deploy:
    block_on:
      - unknown_failure_domains
      - prod_incidents_without_tests
      - missing_error_visibility
  test:
    required:
      - unit
      - integration
      - smoke
      - agentic_manual_execution
  interview:
    developer_questions:
      - "What's the one area you're afraid to touch?"
      - "What broke in production in the last 90 days that tests missed?"
    em_questions:
      # highlight-next-line
      - "Which feature has been blocked for over a year?"
```

:::warning[Friday Deploys Tell You Something]
If Friday deploys never happen, your release confidence is already shot. Measure "time-to-safe-deploy" weekly and tie it to one concrete fix: either remove a flaky test cluster or add production error visibility for one critical path.
:::

## Infrastructure and Runtime Improvements

Cloudflare's ARR and QUIC proxy work solves real routing problems: less NAT pain and measurable throughput/latency gains. PHP runtime updates (JIT support and SQL Server connectivity improvements for runtime generation 2 / 8.2+) matter for enterprise shops running mixed stacks where these friction points slow teams down daily.

```diff title="network-path.diff"
- return traffic resolved by static routing/NAT exceptions
+ return traffic mapped by stateful flow tracking (ARR)
- proxy mode on user-space TCP stack
+ proxy mode on QUIC streams
+ observed impact: higher throughput, lower latency, fewer overlap incidents
```

```bash title="perf-smoke.sh" showLineNumbers
#!/usr/bin/env bash
set -euo pipefail

TARGET_URL="${1:-https://example.internal}"
CONN=20
DURATION=30s

echo "Running HTTP baseline..."
# highlight-next-line
vegeta attack -duration="$DURATION" -connections="$CONN" -targets=targets.txt | vegeta report

echo "Running QUIC-enabled client test..."
cloudflared access tcp --hostname "$TARGET_URL" --url localhost:8080 >/tmp/quic.log 2>&1 &
PID=$!
trap 'kill $PID' EXIT

vegeta attack -duration="$DURATION" -connections="$CONN" -targets=targets.txt | vegeta report
```

## Security: KEV Additions, ICS RCE, and Leaked Keys

CISA added five actively exploited vulnerabilities to KEV this week. That's patch-now intelligence, not a newsletter headline to skim past. Delta CNCSoft-G2 has a remote code execution flaw affecting critical manufacturing environments — internet-facing or bridged OT assets need explicit isolation and compensating controls today, not after quarterly planning wraps up.

GitGuardian and Google's certificate/private-key leakage analysis drives a concrete point home: leaked credentials are not hypothetical debt. Valid certs stay active long enough to be weaponized.

:::danger[Do This in the Next 24 Hours]
Build a KEV-to-asset match for internet-facing systems, revoke/rotate any exposed private keys tied to active certs, and enforce cert inventory ownership. No owner means no accountability means repeat incidents.
:::

## Bottom Line

The pattern that keeps working is unglamorous: ship with explicit gates, patch fast on supported lines, treat AI as a controlled subsystem, and run security operations from exploited-intel feeds instead of quarterly scan summaries.

:::tip[Single Highest-ROI Move]
Add one `release-gates.yaml` policy to CI this week that blocks deployment on: unsupported framework versions, unowned critical certs, and missing executed smoke tests for AI-assisted code paths.
:::
